Manager's Guide

How to manage service connections safely.

1. Service Details

Service Name: The public name your customers will see (e.g., "AI Writer").

Slug: A unique ID used by the system. ⚠️ Do not change this unless you know exactly what you are doing. Changing it can break the app.

Price/Request: How much we charge per usage (e.g., $0.05 per generated document).

2. Webhooks (Automation)

This is the "Brain" connection. It tells the system where to send data to get work done (usually to n8n).

Key: The internal action name (e.g., 'generate').

URL: The web address of the n8n workflow.

Always "Save" individual items. You will be asked for a verification code to confirm changes.

3. API Paths

These are the "Doors" developers access. They are usually auto-discovered from the code.

  • Billable: Check this if users should be charged for using this door.
  • Read-Only: Most paths are set by the developers and cannot be deleted here.

4. Dependencies

Other tools this service relies on.

Example: The "AI Generator" might need the "HTML to PDF" tool to finish its job.

Safety First

This page is protected by 2-Factor Authentication. You must verify every save action to prevent accidental breakages.
