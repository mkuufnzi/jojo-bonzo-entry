
# Floovioo Commercial Model

## 1. Subscription Structure
The platform operates on a tiered subscription model enforced via `SubscriptionService` and `UsageService`.

### Plans (`SubscriptionPlan`)
Defined in database/seeds and `subscription.controller.ts`.
*   **Free**: Basic access, limited quotas (e.g., 5 AI Docs, 15 PDFs).
*   **Pro / Business**: Higher quotas, access to premium features (Brand Engine, n8n Sync).
*   **Enterprise**: Custom quotas, white-glove onboarding.

### Quotas & Limits
Quotas are tracked monthly and reset on the billing cycle.
*   **PDF Quota**: usage tracked via `pdf_conversion` feature key.
*   **AI Quota**: usage tracked via `ai_generation` feature key.
*   **Safety Limits**:
    *   `pdfLimit`: Default 15 (Free).
    *   `aiLimit`: Default 5 (Free).

## 2. Usage Tracking (`UsageService`)
Every consumption event is logged to `UsageLog`.
*   **Attributes**: `userId`, `serviceId`, `action`, `cost`, `status`.
*   **Analytics**:
    *   `getMonthlyUsage(userId)`: Total requests.
    *   `getUserCostByService(userId)`: Breakdown of estimated cost vs price.
    *   **Dashboard**: Displays "Usage Distribution" charts (PDF vs AI) and "Recent Activity".

## 3. Billing Flow (Stripe)
Managed via `PaymentController`.
1.  **Setup Intent**: Created when user adds payment method.
2.  **Payment Method**: Stored in Stripe, reference saved in DB.
3.  **Invoicing**:
    *   Generated by Stripe.
    *   Synced to local DB for display in `/dashboard/billing`.
    *   PDF Invoice generation available via `PaymentController.downloadInvoice` (Html-to-PDF).

## 4. API Monetization
Users can issue **API Keys** to use the platform headless.
*   **Legacy**: App-specific keys.
*   **Modern**: Dedicated API Keys (`sk_live_...`).
*   **Rate Limiting**: Enforced via Redis based on Plan Tier.
